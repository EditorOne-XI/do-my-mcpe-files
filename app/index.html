    What is an Import Builder?

    An Import Builder in the context of Minecraft is a method of importing Minecraft packs and files in a single instance when restarting your setup in Minecraft.

    By restarting, your Minecraft files should be backed up first or Minecraft is freshly installed, otherwise you will lose all of your data. If your File Storage in Minecraft is set to "Application" then there is no other way to back up your Minecraft files unless you have a desktop computer to retrieve your Minecraft files from the system's app data. You can refer to Minecraft File Storage Location.



    Accepted Files to Import

    Most common Minecraft files like .mcpack, .mcaddon, .mcworld, and .mctemplate are completely valid, but there are other Minecraft files supported by this import builder:

    • .zip - Uses the mcpack/mcaddon reading function, some creators provide ZIP files as an alternative.
    • .mcshots - Used for importing Minecraft screenshots with valid format.
    • options.txt - Used to allow players to keep their Minecraft settings.
    • com.mojang.zip - An entire backup file made by either this Import Builder via Save to Storage or QIB - Export to restore the data into the game.


    Why does it need to clear the app's data?

    Most Android applications do not allow other apps to modify their app files for security reasons and they use the "ownership" method by having a signature to every file or directory the app created to their Android/data/ subdirectories. Shizuku interacts with the system through its daemon to grant other apps permission to perform actions that would normally require root. Clearing the app's data is the only solution to erase all files with or without a signature created by the app.

    This web-based app was tested on an Android 14 device to observe the results of modifying unmodified Android/data/ subdirectories. Android 11 to Android 13 may not have issues with this and should uncheck the Clear App Data checkbox.

    If the app is not opened yet and/or has already cleared its data then unchecking the checkbox is fine.

    Minecraft Bedrock v1.17 and below should use Save to Storage instead because the data files are usually exposed in games/ directory if your Minecraft File Storage Location is set to "External". This change was introduced in the Caves and Cliffs: Part II update.



    Save to Storage

    Choosing this option will save the archive to your Download/ directory as com.mojang.zip file. Only use this option if you want to keep a backup of the build.



    Third Party Apps

    Third-party apps for Minecraft also utilize Minecraft's file structure because they use Minecraft itself but with modified game behavior based on how the developer intended the app to be used.

    For a detailed procedure of how to make use of imports to Third Party Apps, please refer to the Third Party Apps guide.
